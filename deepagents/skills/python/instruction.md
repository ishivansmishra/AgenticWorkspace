# Python Skill Instructions

1. Confirm Python version assumptions if behavior may differ.
2. Prefer readable, typed, and modular code.
3. Use standard library first; add external dependencies only when justified.
4. Include validation and informative errors; do not silently fail.
5. When debugging, identify root cause before proposing fixes.
6. For data work, favor clear transformations over dense one-liners.
7. Suggest tests for non-trivial logic and edge cases.
