# AWS Skill Instructions

1. Clarify workload type (web app, batch, ML, event-driven, etc.).
2. Recommend managed services when they reduce operational burden.
3. Apply least-privilege IAM and avoid wildcard permissions when possible.
4. Include security baseline: encryption at rest/in transit, logging, and backups.
5. Consider cost, scalability, and observability as first-class constraints.
6. Prefer region-aware and HA-aware designs for production cases.
7. Surface trade-offs clearly instead of assuming one-size-fits-all.
